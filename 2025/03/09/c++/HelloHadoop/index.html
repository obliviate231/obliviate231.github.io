MapReduce 编程实现单词计数实验报告

一、实习目的和实习意义

  1. 实习目的：深入理解 MapReduce 编程模型的原理和工作机制，熟练掌握使用 MapReduce 进行数据处理的方法，通过完成单词计数这一经典任务，学会在 Hadoop 集群环境中进行代码编写、调试和运行，提升在大数据处理领域的实践操作能力，为今后从事大数据相关工作积累宝贵经验。
  2. 实习意义：在大数据时代，海量数据的处理是关键问题。MapReduce 作为一种高效的分布式计算模型，广泛应用于大数据处理场景。通过本次实习，能够将理论知识与实际操作相结合，体会 MapReduce 在处理大规模文本数据时的优势和重要性。同时，也有助于培养学生的问题解决能力和团队协作精神，使其更好地适应大数据开发岗位的需求，为推动大数据技术的应用和发展贡献力量。

二、实习单位和实习岗位

  1. 实习单位：重庆邮电大学
  2. 实习岗位：大数据开发

三、实习内容和实习过程

（一）实习内容

  1. 搭建和启动 Hadoop 集群环境，确保各节点正常运行。
  2. 在 Eclipse 开发环境中配置 Hadoop 相关依赖，创建 Java 项目用于 MapReduce 编程。
  3. 编写 MapReduce 程序，包括 Mapper 类、Reducer 类和主类，实现单词计数的功能。
  4. 将编写好的程序打包成 JAR 文件，并上传到 Hadoop 集群中运行。
  5. 对运行结果进行验证和分析，查看单词计数的准确性和有效性。

（二）实习过程

  1. 启动 Hadoop 集群
    • 在 master 节点的终端中，输入 [zkpk@master ~]$ start-all.sh 命令启动 Hadoop 集群。这一步骤会启动 Hadoop 的各个组件，包括 NameNode、DataNode、ResourceManager 和 NodeManager 等。
    • 在 master 节点上，使用 jps 命令检查 NameNode、SecondaryNameNode 和 ResourceManager 进程是否成功启动。若这些进程未启动，需要检查 Hadoop 的配置和启动日志，查找并解决问题。
    • 分别在 slave01 和 slave02 节点上运行 jps 命令，确认 DataNode 和 NodeManager 进程已启动。只有当所有节点的相关进程都正常运行时，Hadoop 集群才能正常工作。
  2. 配置 Eclipse 开发环境
    • 进入 Eclipse 安装目录，执行 [zkpk@master ~]$ cd eclipse/ 和 [zkpk@master eclipse]$./eclipse 命令启动 Eclipse 客户端。
    • 在 Eclipse 主界面左侧空白处右击，选择 New->Project，在弹出的项目类型选择窗口中，选择 Java Project 并点击 Next。
    • 在弹出的对话框中，将工程名定义为 hadoop，然后点击 Finish 完成项目创建。当弹出询问是否将 Java Project 设置为常用工程项目时，点击 Yes。
    • 在工程名上单击鼠标右键，依次选择 Build Path ->Configure Build Path。在弹出的窗口中，进入 Libraries 模块，点击 Add External Jars，将 Hadoop 主目录中 share/hadoop/common 下的所有 JAR 包导入。接着再次点击 Add External JARs，将 share/hadoop/common/lib 文件夹中的所有 JAR 包导入。
    • 按照同样的方法，分别将 share/hadoop/hdfs 及其 lib 文件夹、share/hadoop/mapreduce 及其 lib 文件夹、share/hadoop/yarn 及其 lib 文件夹中的 JAR 包导入到项目中。完成所有 JAR 包的导入后，在 Build Path 窗口中点击 OK 按钮，完成开发环境的配置。
  3. 编写 MapReduce 程序
    • 创建 Mapper 类：在工程目录的 src 文件夹上右击，依次选择 New->Package，创建名为 org.zkpk.hadoop.wordcount 的包。在该包上右键点击，选择 New->Class，创建名为 WordMap 的类。让 WordMap 类继承 Mapper 类，定义输入键值对的键为 Object 类型，值为 Text 类型；输出键值对的键为 Text 类型，值为 IntWritable 类型。覆写 Mapper 类中的 map 方法，在方法中，将输入的 value 转换成 String 类型，按照字段之间的分隔符进行切分，得到字符串数组。使用 for 循环遍历单词数组，将每次遍历得到的单词包装成 Text 类型，将数字 1 包装成 IntWritable 类型，分别作为输出的键和值。
    • 创建 Reducer 类：在 org.zkpk.hadoop.wordcount 包上右键点击，依次选择 New->Class，创建名为 WordReduce 的类。让 WordReduce 类继承 Reducer 类，定义输入键值对的键为 Text 类型，值为 IntWritable 类型（与 Mapper 输出的键值对类型一致）；输出键值对的键为 Text 类型，值为 IntWritable 类型。覆写 Reducer 类中的 reduce 方法，在方法中，reduce 会将相同 key 的键值对汇聚到一个 reduce task 中。第一个参数 key 表示单词，第二个参数 Values 表示当前 key 对应的所有 value 集合。对 values 进行遍历，将每次遍历的数值累加到变量 sum 中。遍历结束后，将参数 key 作为输出键值对中的键，将 sum 包装成 IntWritable 类型作为输出键值对中的值，使用 context 的 write 方法写出。
    • 创建包含 main 方法的类：在 org.zkpk.hadoop.wordcount 包上右键点击，依次选择 New->Class，创建名为 WordMain 的类。在 WordMain 类中创建 main 方法，在方法中使用 if 语句判断传入的参数格式是否正确（本程序需要接受两个参数作为输入和输出路径），如果不正确则退出程序。生成 Configuration 类型对象，利用该对象生成 job 对象。调用 job 的 setJarByClass 方法设置 job 运行主类，通过 setInputFormatClass 方法设置 job 输入格式，setOutputFormatClass 方法设置 job 输出格式。通过 setInputPaths 和 setOutputPath 方法设置 job 的输入、输出路径，通过 setMapperClass 和 setReducerClass 方法设置 Map/Reduce 阶段的类，通过 setOutputKeyClass 和 setOutputValueClass 方法设置最终输出 key/value 的类型。最后，通过 waitForCompletion 方法提交 job。
  4. 程序打 JAR 包
    • 右击工程名，点击 export，在弹出的窗口中，选择 java 的 JAR file 并点击 Next。
    • 在弹出的对话框中，指定导出的 JAR 包的位置及名字，然后点击 Finish。
    • 导出完成后，关闭 Eclipse 并返回指定目录查看生成的 JAR 文件。
  5. 运行 JAR 包并查看结果
    • 上传输入数据：从 Hadoop 的公共目录下拷贝数据文件 test.txt 到 /home/zkpk，执行 [zkpk@master ~]$ cp ~/experiment/test.txt ~/。然后将拷贝的数据文件 test.txt 上传到 HDFS 的根目录，执行 [zkpk@master ~]$ hadoop fs -put /home/zkpk/test.txt /。
    • 查看上传结果：使用 [zkpk@master ~]$ hadoop fs -ls / 命令查看文件是否成功上传到 HDFS 根目录。
    • 运行 JAR 包：运行 JAR 包，指定包名及主类名，并指定输入路径参数和输出路径参数（输出路径即 word 文件夹不能够已存在），执行 [zkpk@master ~]$ hadoop jar /home/zkpk/wordcount.jar org.zkpk.hadoop.wordcount.WordMain /test.txt /word。
    • 查看输出目录和结果：使用 [zkpk@master ~]$ hadoop fs -ls /word/ 命令查看输出目录，使用 [zkpk@master ~]$ hadoop fs -cat /word/part-r-00000 命令查看输出结果文件，检查单词计数的结果是否正确。

四、总结与体会

通过本次实习，我对 MapReduce 编程模型有了更深入的理解和掌握。从搭建 Hadoop 集群到配置开发环境，再到编写、打包和运行 MapReduce 程序，每一个环节都让我体会到大数据处理的复杂性和严谨性。在编写代码的过程中，对 Mapper 和 Reducer 类的设计和实现有了更清晰的思路，理解了它们在分布式计算中的作用和协作方式。同时，也学会了如何在 Hadoop 集群上部署和运行程序，并对运行结果进行分析和验证。

在实习过程中，也遇到了一些问题，比如 Hadoop 集群启动失败、代码编译错误等。通过查阅相关文档和资料，逐步解决了这些问题，这不仅提高了我的问题解决能力，也增强了我的自信心。此外，我还深刻体会到团队协作的重要性，在与同学交流和讨论的过程中，能够获得更多的思路和方法，提高工作效率。

总的来说，这次实习让我收获颇丰，不仅掌握了 MapReduce 编程技能，也对大数据开发有了更深刻的认识。在今后的学习和工作中，我将继续努力，不断提升自己的专业能力，为大数据领域的发展贡献自己的力量。
